### 1.3 系统角色

| 角色名称 | 角色标识 | 权限范围 | 说明 |
|---------|---------|---------|------|
| 超级管理员 | admin | 全部功能 | 拥有系统所有权限，可管理用户、角色、菜单等 |
| 普通管理员 | manager | 部分管理功能 | 可管理用户、内容审核，但不能管理角色和菜单 |
| 内容创作者 | creator | 内容管理 | 可发布和管理内容、帖子，查看互动数据 |
| 普通用户 | user | 前端功能 | 仅能访问小程序端，不能登录后台管理系统 |

### 1.4 系统架构

#### 前后端分离架构
- **前端**：Vue3 + Element Plus，运行在端口3000
- **后端**：Spring Boot，运行在端口8080，提供RESTful API
- **数据库**：MySQL 8.0+，数据库名hotai_db
- **通信方式**：前端通过Axios发送HTTP请求，后端返回JSON格式数据
- **认证方式**：JWT Token认证，Token存储在localStorage

#### 开发环境配置
- **前端开发地址**：http://localhost:3000
- **后端API地址**：http://localhost:8080/api
- **Swagger文档地址**：http://localhost:8080/api/swagger-ui.html
- **数据库连接**：jdbc:mysql://localhost:3306/hotai_db

### 1.6 项目特色功能

#### 技术亮点
1. **模块化架构**：API、Mock、Store、Router完全模块化设计
2. **统一请求封装**：Axios拦截器统一处理Token、错误提示
3. **路由守卫**：基于Token和角色的身份认证和权限控制
4. **动态路由**：支持根据角色权限动态生成菜单
5. **MyBatis-Plus集成**：简化CRUD操作，支持分页、逻辑删除
6. **JWT认证**：无状态认证，支持Token过期自动登出
7. **操作日志AOP**：基于注解的操作日志自动记录
8. **JSON字段支持**：MySQL JSON类型字段的自动映射
9. **BCrypt密码加密**：安全的密码存储方案
10. **Swagger文档**：自动生成API文档，方便接口测试

#### 业务特色
1. **内容统一管理**：文章、帖子统一存储在content表，通过类型区分
2. **多级评论系统**：支持评论和回复的树形结构
3. **互动数据统计**：点赞、收藏、分享数据的完整记录和统计
4. **内容审核流程**：支持内容的待审核、通过、拒绝状态管理
5. **Feed流推荐**：支持内容推荐算法配置
6. **用户标签系统**：支持用户兴趣标签管理和推荐
7. **角色权限控制**：细粒度的角色权限管理
8. **操作日志追踪**：完整记录用户操作行为

### 1.7 开发规范

#### 命名规范
- **文件名**：小驼峰命名（user.js, userTag.vue）
- **组件名**：大驼峰命名（UserTag, ContentList）
- **路由name**：大驼峰命名（User, ContentList）
- **API方法**：小驼峰命名（getUserList, addUser）
- **数据库表名**：下划线命名（sys_user, content_category）
- **Java类名**：大驼峰命名（SysUser, ContentController）

#### 代码规范
- 使用ES6+语法
- 使用async/await处理异步
- 统一使用箭头函数
- API调用统一使用try-catch
- 组件使用Composition API（推荐）
- Java使用Lombok简化代码
- 使用MyBatis-Plus的BaseMapper和ServiceImpl

#### 接口规范
- 统一响应格式：`{code: 200, message: "成功", data: {}}`
- RESTful风格：GET查询、POST新增、PUT更新、DELETE删除
- 分页参数：pageNum（页码）、pageSize（每页数量）
- 分页响应：`{total: 总数, list: 数据列表}`
