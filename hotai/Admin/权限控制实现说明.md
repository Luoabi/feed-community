# 后台管理系统权限控制实现说明

## 一、权限控制规则

### 1. 登录权限控制

#### ❌ 普通用户（role_id=4）禁止登录后台
- **实现位置**：后端 `SysUserController.login()`
- **验证逻辑**：检查用户角色，如果只有"普通用户"角色，拒绝登录
- **错误提示**：`普通用户无权限登录后台管理系统`

#### ✅ 允许登录的角色
- 超级管理员（role_id=1）
- 普通管理员（role_id=2）
- 内容创作者（role_id=3）
- 拥有多个角色的用户（如：普通用户+内容创作者）

### 2. 菜单权限控制

#### 前端路由权限配置

所有路由在 `router/index.js` 中配置 `meta.roles` 字段：

```javascript
{
  path: '/system',
  meta: { 
    roles: ['超级管理员', '普通管理员'] 
  }
}
```

#### 侧边栏菜单过滤

`Sidebar.vue` 组件根据用户角色自动过滤菜单：
- 只显示用户有权限的菜单
- 无权限的菜单不显示

#### 路由守卫验证

`router.beforeEach` 守卫验证：
- 用户访问无权限路由时，自动跳转到首页
- 防止通过URL直接访问无权限页面

## 三、实现细节

### 1. 后端登录验证

**文件**：`hotaihoduan/BRApi/src/main/java/org/xingchang/brapi/controller/SysUserController.java`

```java
// ✅ 检查用户角色：普通用户不能登录后台管理系统
if (user.getRoleIds() != null && 
    user.getRoleIds().contains(4L) && 
    user.getRoleIds().size() == 1) {
    return Result.error("普通用户无权限登录后台管理系统");
}
```

**逻辑说明**：
- 检查用户的 `roleIds` 是否包含 `4L`（普通用户）
- 检查用户是否**只有**普通用户角色（`size() == 1`）
- 如果是纯普通用户，拒绝登录
- 如果用户有多个角色（如：普通用户+内容创作者），允许登录

---

### 2. 前端路由权限配置

**文件**：`hotai/Admin/src/router/index.js`

#### 路由配置示例

```javascript
{
  path: '/system',
  meta: { 
    title: '系统管理',
    roles: ['超级管理员', '普通管理员'] // 只有这两个角色可以访问
  },
  children: [
    {
      path: 'role',
      meta: { 
        title: '角色管理',
        roles: ['超级管理员'] // 只有超级管理员可以访问
      }
    }
  ]
}
```

#### 权限继承规则

1. **父路由有roles，子路由无roles**：子路由继承父路由权限
2. **父路由有roles，子路由也有roles**：子路由使用自己的权限（更严格）
3. **父路由无roles，子路由有roles**：子路由使用自己的权限

### 3. 侧边栏菜单过滤

**文件**：`hotai/Admin/src/layout/components/Sidebar.vue`

```javascript
// 根据用户角色过滤菜单
const routes = computed(() => {
  const userRoles = store.state.user.roles || []
  
  const allRoutes = router.options.routes.filter(r => {
    return r.path !== '/login' && 
           r.path !== '/404' && 
           r.children && 
           r.children.length > 0
  })
  
  // 根据角色过滤路由
  return allRoutes.filter(route => {
    return hasPermission(route, userRoles)
  })
})

function hasPermission(route, userRoles) {
  // 如果路由没有定义roles，则所有登录用户都可以访问
  if (!route.meta || !route.meta.roles) {
    return true
  }
  
  // 检查用户角色是否在路由允许的角色列表中
  return userRoles.some(role => route.meta.roles.includes(role))
}
```

**逻辑说明**：
- 获取用户的角色列表
- 遍历所有路由，检查用户是否有权限
- 只返回有权限的路由
- 侧边栏只显示过滤后的菜单

---

### 4. 路由守卫验证

**文件**：`hotai/Admin/src/router/index.js`

```javascript
router.beforeEach((to, from, next) => {
  const token = store.state.user.token
  const userRoles = store.state.user.roles || []
  
  // 需要认证的路由
  if (to.meta.requiresAuth && !token) {
    next('/login')
    return
  }
  
  // 已登录访问登录页，跳转首页
  if (to.path === '/login' && token) {
    next('/')
    return
  }
  
  // ✅ 检查路由角色权限
  if (to.meta.roles && to.meta.roles.length > 0) {
    const hasRole = userRoles.some(role => to.meta.roles.includes(role))
    if (!hasRole) {
      // 无权限，跳转到首页
      next('/')
      return
    }
  }
  
  next()
})
```

**逻辑说明**：
- 检查用户是否登录
- 检查路由是否需要特定角色
- 验证用户角色是否匹配
- 无权限时跳转到首页（防止直接访问URL）

## 六、常见问题

### Q1: 用户有多个角色怎么办？

**A**: 系统支持多角色，只要用户拥有的任一角色满足路由要求，就可以访问。

例如：用户同时是"普通用户"和"内容创作者"，可以登录后台，并访问内容创作者的所有菜单。

---

### Q2: 如何添加新的角色？

**A**: 
1. 在数据库 `sys_role` 表添加新角色
2. 在前端路由配置中添加角色名称
3. 重新分配菜单权限

---

### Q3: 如何修改某个菜单的权限？

**A**: 修改 `router/index.js` 中对应路由的 `meta.roles` 字段

```javascript
{
  path: 'audit',
  meta: { 
    roles: ['超级管理员', '普通管理员', '内容创作者'] // 添加内容创作者
  }
}
```
