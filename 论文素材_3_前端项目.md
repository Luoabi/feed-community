# 项目技术素材 - 第3部分：前端项目结构分析

## 三、前端项目结构分析（Vue3）

### 3.1 目录结构

| 目录 | 用途 |
|------|------|
| src/views | 页面组件（20个页面） |
| src/components | 公共组件（布局组件） |
| src/router | 路由配置（动态路由、路由守卫） |
| src/api | 接口请求封装（14个API模块） |
| src/store | 状态管理（3个Store模块） |
| src/mock | Mock数据（14个Mock模块） |
| src/layout | 布局组件（主布局、侧边栏、导航栏） |
| src/styles | 全局样式 |
| src/utils | 工具函数（日志工具） |
| src/assets | 静态资源（图片、图标） |
| public | 公共静态资源 |

| /404 | NotFound | 404错误页 | - | 无需认证 |

#### 路由守卫逻辑
1. 检查路由是否需要认证（meta.requiresAuth）
2. 未登录且需要认证，跳转到登录页
3. 已登录访问登录页，跳转到首页
4. 检查路由角色权限（meta.roles）
5. 用户角色不匹配，跳转到首页或403页面
6. 权限验证通过，允许访问

### 3.5 状态管理（Vuex）

#### user模块（用户状态）
| 状态字段 | 类型 | 说明 | 持久化 |
|---------|------|------|--------|
| token | String | 用户Token | 是（localStorage） |
| userInfo | Object | 用户信息 | 是（localStorage） |
| roles | Array | 用户角色列表 | 否 |

**Actions:**
- login(username, password)：用户登录，保存Token和用户信息
- logout()：用户登出，清除所有状态
- getUserInfo()：获取用户信息

#### app模块（应用状态）
| 状态字段 | 类型 | 说明 |
|---------|------|------|
| sidebar | Object | 侧边栏状态（展开/收起） |
| device | String | 设备类型（desktop/mobile） |
| size | String | 组件尺寸（large/default/small） |

#### permission模块（权限状态）
| 状态字段 | 类型 | 说明 |
|---------|------|------|
| routes | Array | 所有路由 |
| addRoutes | Array | 动态添加的路由 |

**Actions:**
- generateRoutes(roles)：根据角色生成可访问路由

### 3.7 请求拦截器功能

#### 请求拦截器（request.js）
1. 自动添加Authorization Token到请求头
2. 设置请求超时时间（15秒）
3. 统一处理请求参数格式

#### 响应拦截器（request.js）
1. 统一处理响应数据格式
2. Token过期（code: 401）自动登出
3. 错误提示（使用Element Plus的Message组件）
4. 统一错误处理和消息提示
