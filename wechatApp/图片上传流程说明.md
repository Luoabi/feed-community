# 图片上传到后端的完整流程

## 📸 流程概述

### 1. 前端上传图片
```javascript
// 用户选择图片
wx.chooseImage({
  count: 9,
  success: (res) => {
    const tempFilePaths = res.tempFilePaths
    // tempFilePaths: ["wxfile://tmp_xxx.jpg", "wxfile://tmp_yyy.jpg"]
  }
})
```

### 2. 上传到后端服务器
```javascript
// 使用 wx.uploadFile 上传每张图片
wx.uploadFile({
  url: 'http://localhost:8080/api/upload/image',
  filePath: 'wxfile://tmp_xxx.jpg',
  name: 'file',
  header: { 'token': token },
  success: (res) => {
    const data = JSON.parse(res.data)
    // data.data: "http://localhost:8080/api/uploads/images/2026/04/12/xxx.jpg"
  }
})
```

### 3. 后端处理上传
```java
@PostMapping("/image")
```

### 4. 收集所有图片URL
```javascript
// 批量上传，收集所有URL
const uploadedImages = await this.uploadImages()
// uploadedImages: [
//   "http://localhost:8080/api/uploads/images/2026/04/12/xxx.jpg",
//   "http://localhost:8080/api/uploads/images/2026/04/12/yyy.jpg"
// ]
```

### 5. 发布内容时传递URL数组
```javascript
const contentData = {
  title: '标题',
  content: '内容',
  contentType: 'image',
  images: uploadedImages,  // ✅ 直接传数组
  authorId: userInfo.id,
  authorName: userInfo.nickname
}

await api.publishContent(contentData)
```

### 6. 后端存储到数据库
```java
@Data
@TableName("content")

    // ✅ 使用 JacksonTypeHandler 自动转换 List<String> ↔ JSON
    @TableField(typeHandler = JacksonTypeHandler.class)

```

**数据库存储格式：**
```json
["http://localhost:8080/api/uploads/images/2026/04/12/xxx.jpg","http://localhost:8080/api/uploads/images/2026/04/12/yyy.jpg"]
```

## 🔧 关键技术点

### 1. MyBatis-Plus 类型处理器
```java
@TableField(typeHandler = JacksonTypeHandler.class)
private List<String> images;
```
- **作用**：自动将 `List<String>` 转换为 JSON 字符串存储
- **存储时**：`["url1", "url2"]` → `'["url1","url2"]'`
- **读取时**：`'["url1","url2"]'` → `["url1", "url2"]`

### 2. 文件命名规则
```java
// UUID + 扩展名
String newFilename = UUID.randomUUID().toString().replace("-", "") + extension;
// 示例：66595247bdc645988d251e55fda30748.jpg
```

### 3. 目录结构
```
uploads/
  ├── images/
  │   ├── 2026/
  │   │   ├── 04/
  │   │   │   ├── 12/
  │   │   │   │   ├── xxx.jpg
  │   │   │   │   ├── yyy.jpg
  ├── videos/
  │   ├── 2026/
  │   │   ├── 04/
  │   │   │   ├── 12/

  ├── avatars/
      ├── 2026/

```

### 4. URL 格式
```

│                        │      │      │    │  │    │
└─ 域名                   └─ 类型 └─ 年  └─月 └─日  └─ UUID文件名
```

## 📝 代码示例

### 前端：批量上传图片
```javascript
uploadImages() {
  return new Promise((resolve, reject) => {
    const token = wx.getStorageSync('token')
    const uploadPromises = this.data.images.map((imagePath) => {
      return new Promise((resolveUpload, rejectUpload) => {
        wx.uploadFile({
          url: 'http://localhost:8080/api/upload/image',
          filePath: imagePath,
          name: 'file',
          header: { 'token': token },
          success: (res) => {
            const data = JSON.parse(res.data)
            if (data.code === 200) {
              resolveUpload(data.data)  // 返回URL
            } else {
              rejectUpload(new Error(data.message))
            }
          },
          fail: (err) => rejectUpload(err)
        })
      })
    })
    
    Promise.all(uploadPromises)
      .then(urls => resolve(urls))  // 返回URL数组
      .catch(err => reject(err))
  })
}
```

### 前端：发布内容
```javascript
async onPublish() {
  // 1. 上传图片
  const uploadedImages = await this.uploadImages()
  
  // 2. 发布内容
  const contentData = {
    title: this.data.title,
    content: this.data.content,
    contentType: 'image',
    images: uploadedImages,  // ✅ URL数组
    authorId: userInfo.id,
    authorName: userInfo.nickname
  }
  
  await api.publishContent(contentData)
}
```

### 后端：保存文件
```java
private String saveFile(MultipartFile file, String subDir) throws IOException {
    // 1. 生成文件名
    String newFilename = UUID.randomUUID().toString().replace("-", "") + extension;
    
    // 2. 创建目录
    String dateDir = LocalDate.now().format(DateTimeFormatter.ofPattern("yyyy/MM/dd"));
    String relativePath = subDir + "/" + dateDir + "/" + newFilename;
    Path uploadDir = uploadBasePath.resolve(subDir).resolve(dateDir);
    Files.createDirectories(uploadDir);
    
    // 3. 保存文件
    Path filePath = uploadDir.resolve(newFilename);
    file.transferTo(filePath.toFile());
    
    // 4. 返回URL
    return uploadDomain + "/uploads/" + relativePath;
}
```

## ✅ 总结

### 关键点
1. **前端上传** → 每张图片单独上传，获得URL
2. **收集URL** → 将所有URL放入数组
3. **传递数组** → 发布时直接传 `images: [url1, url2]`
4. **自动转换** → 后端使用 `JacksonTypeHandler` 自动转换
5. **JSON存储** → 数据库以JSON字符串格式存储

### 优势
- ✅ 前端逻辑简单，只需收集URL
- ✅ 后端自动处理JSON转换
- ✅ 数据库存储紧凑
- ✅ 查询时自动反序列化为List

### 注意事项
- 🔒 上传时需要验证Token
- 📏 限制文件大小（图片5MB，视频50MB）
- 🎨 限制文件类型（只允许图片/视频）
- 📁 按日期分目录存储，便于管理
- 🔐 使用UUID命名，避免文件名冲突
