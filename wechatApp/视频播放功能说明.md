# 小程序首页视频播放功能说明

## 功能概述

首页的视频内容现在支持直接播放，无需跳转到详情页。同时，点击视频下方的标题、作者信息等区域仍然可以跳转到详情页查看完整内容。

## 功能特性

### 1. 视频直接播放
- ✅ 视频内容在首页列表中直接显示 `<video>` 组件
- ✅ 用户可以直接点击播放按钮观看视频
- ✅ 支持全屏播放
- ✅ 支持进度条拖动
- ✅ 显示视频时长（如果有）

### 2. 智能播放控制
- ✅ 同一时间只能播放一个视频（新视频播放时自动暂停其他视频）
- ✅ 切换分类时自动暂停所有视频
- ✅ 页面隐藏时自动暂停所有视频
- ✅ 下拉刷新时自动暂停所有视频

### 3. 跳转详情页
- ✅ 点击视频区域：播放/暂停视频（不跳转）
- ✅ 点击标题区域：跳转到详情页
- ✅ 点击作者信息：跳转到详情页
- ✅ 点击统计信息：跳转到详情页

### 4. 错误处理
- ✅ 视频加载失败时显示错误提示
- ✅ 视频地址无效时显示占位图和提示信息

## 代码结构

### WXML 结构
```xml
<view class="video-wrapper">
  <!-- 视频播放区域（阻止冒泡，不跳转） -->
  <view class="video-container" catchtap="stopPropagation">
    <video id="video-{{item.id}}" src="{{item.videoUrl}}" ... />
  </view>
  
  <!-- 信息区域（可点击跳转） -->
  <view class="content-info" bindtap="onContentTap" data-id="{{item.id}}">
    <view class="content-title">{{item.title}}</view>
    ...
  </view>
</view>
```

### JS 事件处理
```javascript
// 视频播放事件
onVideoPlay(e) {
  const videoId = e.currentTarget.dataset.id
  // 暂停其他正在播放的视频
  if (this.data.currentPlayingVideoId && this.data.currentPlayingVideoId !== videoId) {
    const videoContext = wx.createVideoContext(`video-${this.data.currentPlayingVideoId}`, this)
    videoContext.pause()
  }
  this.setData({ currentPlayingVideoId: videoId })
}

// 跳转详情页
onContentTap(e) {
  const id = e.currentTarget.dataset.id
  // 暂停当前播放的视频
  if (this.data.currentPlayingVideoId) {
    const videoContext = wx.createVideoContext(`video-${this.data.currentPlayingVideoId}`, this)
    videoContext.pause()
  }
  wx.navigateTo({ url: `/pages/content/detail?id=${id}` })
}

// 阻止事件冒泡
stopPropagation(e) {
  // 视频区域不跳转到详情页
}
```

## 使用方式

### 用户操作
1. **播放视频**：点击视频中央的播放按钮或视频任意位置
2. **暂停视频**：再次点击视频
3. **全屏播放**：点击视频右下角的全屏按钮
4. **查看详情**：点击视频下方的标题、作者或统计信息

### 开发者配置
确保内容数据包含以下字段：
```javascript
{
  id: 1,
  contentType: 'video',
  title: '视频标题',
  videoUrl: 'https://example.com/video.mp4',  // 必需
  coverImage: 'https://example.com/cover.jpg', // 视频封面
  videoDuration: '03:25',  // 可选，视频时长
  authorName: '作者名称',
  authorAvatar: 'https://example.com/avatar.jpg',
  views: 1234,
  likes: 56
}
```

## 注意事项

1. **视频格式**：建议使用 MP4 格式，H.264 编码
2. **视频大小**：建议视频文件不超过 10MB，以保证加载速度
3. **网络环境**：视频播放需要良好的网络环境
4. **HTTPS**：生产环境必须使用 HTTPS 协议的视频地址
5. **域名配置**：需要在小程序后台配置视频域名白名单

## 性能优化

1. **懒加载**：视频组件使用懒加载，滚动到可视区域才加载
2. **单例播放**：同一时间只播放一个视频，节省资源
3. **自动暂停**：页面切换、分类切换时自动暂停视频
4. **封面图**：使用 poster 属性显示视频封面，减少初始加载
