# 头像上传功能测试指南

## 📋 功能概述

用户可以在"编辑资料"页面上传和更换头像，头像会自动上传到后端服务器并保存到数据库。

---

## 🔧 后端接口

### 1. 头像上传接口
```
POST /api/upload/avatar
Content-Type: multipart/form-data
Header: token: <用户token>

参数:
- file: 图片文件

返回:
{
  "code": 200,
  "message": "success",
  "data": "http://localhost:8080/api/uploads/avatars/2026/04/12/xxx.jpg"
}
```

### 2. 更新用户信息接口
```
PUT /api/user/update
Content-Type: application/json
Header: token: <用户token>

参数:
{
  "avatar": "http://localhost:8080/api/uploads/avatars/2026/04/12/xxx.jpg",
  "nickname": "用户昵称",
  "gender": 1,
  "bio": "个人简介",
  "phone": "13800138000",
  "email": "user@example.com"
}

返回:
{
  "code": 200,
  "message": "success"
}
```

## 🐛 常见问题排查

### 问题1: 上传失败 - "请先登录"
**原因**: 没有 token 或 token 过期

**解决**:
1. 检查是否已登录
2. 查看控制台是否有 token
3. 重新登录获取新 token

```javascript
// 检查 token
const token = wx.getStorageSync('token')
console.log('Token:', token)
```

### 问题2: 上传失败 - "网络请求失败"
**原因**: 后端服务未启动或地址错误

**解决**:
1. 确认后端服务已启动
2. 检查 `utils/upload.js` 中的 BASE_URL
3. 确保手机和电脑在同一局域网（真机调试）

```javascript
// utils/upload.js
const BASE_URL = 'http://localhost:8080/api'  // 开发环境
// const BASE_URL = 'http://192.168.1.100:8080/api'  // 真机调试
```

### 问题3: 上传成功但保存失败
**原因**: 更新用户信息接口失败

**解决**:
1. 查看控制台错误信息
2. 检查后端日志
3. 确认用户信息格式正确

```javascript
// 查看保存的数据
console.log('保存数据:', formData)
```

### 问题4: 头像显示不出来
**原因**: 图片URL处理问题或静态资源配置问题

**解决**:
1. 检查返回的URL格式
2. 确认后端静态资源配置
3. 查看 `utils/image.js` 的处理逻辑

```javascript
// 查看头像URL
console.log('头像URL:', userInfo.avatar)
```

### 问题5: 上传的图片太大
**原因**: 图片超过2MB限制

**解决**:
1. 使用压缩后的图片
2. 调整 `sizeType: ['compressed']`
3. 后端调整大小限制

## 🎓 代码说明

### 前端上传流程
```javascript
// 1. 选择图片
wx.chooseImage() 
  → 获得临时文件路径

// 2. 显示临时图片
setData({ avatar: tempFilePath })
  → 用户立即看到选择的图片

// 3. 上传到服务器
uploadAvatar(tempFilePath)
  → 调用后端接口

// 4. 更新为服务器URL
setData({ avatar: serverUrl })
  → 使用服务器返回的URL

// 5. 保存用户信息
api.updateUserInfo({ avatar: serverUrl })
  → 保存到数据库
```

### 后端处理流程
```java
// 1. 接收文件
@PostMapping("/avatar")
uploadAvatar(@RequestParam("file") MultipartFile file)

// 2. 验证文件
- 检查是否为空
- 检查文件类型（只允许图片）
- 检查文件大小（最大2MB）

// 3. 保存文件
- 生成UUID文件名
- 创建日期目录
- 保存到磁盘

// 4. 返回URL
return Result.success(fileUrl)
```

## ✨ 功能特点

1. **即时预览** - 选择图片后立即显示
2. **后台上传** - 不阻塞用户操作
3. **上传状态** - 显示上传进度和结果
4. **错误处理** - 上传失败自动恢复原头像
5. **防重复** - 上传中禁止再次上传
6. **自动保存** - 上传成功后自动更新数据库

---

## 📝 注意事项

1. **图片大小**: 头像限制2MB，普通图片限制5MB
2. **图片格式**: 支持 jpg、png、gif 等常见格式
3. **网络环境**: 真机调试需要同一局域网
4. **Token验证**: 必须登录后才能上传
5. **文件命名**: 使用UUID避免文件名冲突
6. **目录结构**: 按日期分目录便于管理
