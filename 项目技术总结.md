## 一、技术架构总结

### 1.1 整体架构
本项目采用前后端分离架构，前端使用Vue3生态系统，后端使用Spring Boot框架，数据库使用MySQL 8.0。

**架构特点：**
- 前后端完全解耦，独立开发和部署
- RESTful API设计，接口规范统一
- JWT Token无状态认证
- 模块化设计，高内聚低耦合
- 支持水平扩展

### 1.2 技术选型理由

**前端技术选型：**
- Vue3：性能优异，Composition API提供更好的代码组织
- Element Plus：成熟的企业级UI组件库，开箱即用
- Vite：极速的开发体验，HMR响应快
- Vuex：集中式状态管理，适合中大型应用
- Axios：功能强大的HTTP客户端，支持拦截器

**后端技术选型：**
- Spring Boot 3.4.1：最新稳定版，开发效率高
- MyBatis-Plus 3.5.7：简化CRUD操作，提供丰富的增强功能
- JWT 0.12.3：轻量级认证方案，适合前后端分离
- MySQL 8.0：成熟稳定的关系型数据库
- Swagger：自动生成API文档，方便接口测试


## 二、核心技术实现

### 2.1 认证与授权

**JWT Token认证机制：**
- 用户登录成功后生成JWT Token（有效期24小时）
- Token包含userId和username信息
- 前端将Token存储在localStorage
- 每次请求在Header中携带Token
- 后端拦截器验证Token有效性
- Token过期自动跳转登录页

**密码加密：**
- 使用BCrypt算法加密密码
- 每次加密生成不同的盐值
- 密码验证使用BCrypt.matches方法
- 安全性高，防止彩虹表攻击

**角色权限控制：**
- 基于角色的访问控制（RBAC）
- 前端路由守卫检查角色权限
- 后端接口可添加角色验证
- 支持细粒度的权限控制

### 2.2 数据持久化

**MyBatis-Plus增强功能：**
- BaseMapper：提供基础CRUD方法
- ServiceImpl：简化Service层代码
- 分页插件：自动处理分页查询
- 逻辑删除：deleted字段软删除
- 自动填充：创建时间、更新时间
- 条件构造器：灵活的查询条件

**JSON字段处理：**
- 自定义TypeHandler处理MySQL JSON类型
- JsonLongListTypeHandler：处理Long数组
- JsonStringListTypeHandler：处理String数组
- 自动序列化和反序列化

### 2.3 操作日志

**AOP切面实现：**
- 使用@OperationLog注解标记需要记录的方法
- OperationLogAspect切面自动拦截
- 记录操作用户、模块、类型、参数、耗时等
- 异常情况也会被记录
- 支持日志查询、删除、清空

### 2.4 文件上传

**上传功能实现：**
- 支持图片、视频、文件上传
- 文件类型验证
- 文件大小限制（图片5MB，视频50MB）
- 生成唯一文件名（UUID）
- 返回文件访问URL
- 支持多文件上传

## 五、技术难点与解决方案

### 5.1 跨域问题
**问题：** 前端（3000端口）访问后端（8080端口）存在跨域限制

**解决方案：**
- 开发环境：Vite配置proxy代理
- 生产环境：后端配置CORS允许跨域
- 使用CorsConfig配置类统一处理

### 5.2 Token过期处理
**问题：** Token过期后用户体验不佳

**解决方案：**
- 响应拦截器捕获401状态码
- 自动清除本地Token和用户信息
- 跳转到登录页并提示
- 可扩展：实现Token自动刷新机制

### 5.3 JSON字段映射
**问题：** MySQL JSON类型字段无法直接映射到Java对象

**解决方案：**
- 自定义TypeHandler处理JSON字段
- JsonLongListTypeHandler处理Long数组
- JsonStringListTypeHandler处理String数组
- 在Entity字段上添加@TableField注解指定TypeHandler

### 5.4 密码安全
**问题：** 密码明文存储不安全

**解决方案：**
- 使用BCrypt算法加密密码
- 每次加密生成不同的盐值
- 密码验证使用BCrypt.matches
- 前端不传输明文密码（可扩展：前端加密）

### 5.5 大数据量分页
**问题：** 数据量大时分页查询性能下降

**解决方案：**
- 使用MyBatis-Plus分页插件
- 数据库添加索引优化查询
- 避免使用SELECT *
- 可扩展：使用Redis缓存热点数据

### 5.6 文件上传安全
**问题：** 文件上传存在安全风险

**解决方案：**
- 验证文件类型（白名单）
- 限制文件大小
- 生成随机文件名（UUID）
- 文件存储在安全目录
- 可扩展：使用OSS对象存储

## 六、性能优化措施

### 6.1 前端性能优化
- 路由懒加载：按需加载页面组件
- 组件按需引入：减少打包体积
- Vite构建优化：代码分割、压缩
- 图片懒加载：减少首屏加载时间

### 6.2 后端性能优化

- 分页查询：避免一次性加载大量数据
- 逻辑删除：避免物理删除的性能开销
- 连接池配置：优化数据库连接管理
- 可扩展：Redis缓存、消息队列

### 6.3 数据库优化

- 使用EXPLAIN分析查询：优化慢查询
- 字段类型优化：选择合适的数据类型
- 表结构优化：避免冗余字段
